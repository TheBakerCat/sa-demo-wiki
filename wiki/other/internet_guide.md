Ультимативный гайд на настройку интернета/связи между виртальными машинами

## 2.1 Создание и настройка интерфейсов

В директории /etc/net/ifaces создаем(mkdir "имя интерфейса") или копируем(cp ens18 "имя нового интерфейса", если уже существует интерфейс)
По итогу на машине IPS должно получиться 3 интерфейса: ens18 ens19 ens20

Переходим в созданную директорию, создаем файлы(если их нет):

```
touch options ipv4address ipv4route resolv.conf
```

### 2.2 Настройка динамеческого(DHCP) адреса
Этим способом настраиваетсья только ens18(интерфейс в сторону интернета) на машине IPS

Редактируем файл options следуйшим образом:
```
BOOTPROTO=dhcp
TYPE=eth
CONFIG_IPV4=yes
CONFIG_WIRELESS=no
DISABLED=no
```
Больше ничего настраивать не нужно

### 2.3 Настройка статического адреса

Этим способом настриваетсья все остольные машины

Начинаем настраивать файлы следуйщим образом:

Файл options 
```
BOOTPROTO=static
TYPE=eth
CONFIG_IPV4=yes
CONFIG_WIRELESS=no
DISABLED=no
```

Файл ipv4address
```
ip адрес интерфейса + префикс(ака сокрашенная маска),
```
например 172.16.1.1/28 для HQ-RTR и 172.16.2.1/28 для BR-RTR

Файл ipv4route
```
ip адрес шлюза по умолчанию, куда должен вести интерфейс (маску писать не надо),
```

Файл resolv.conf
```
nameserver 8.8.8.8
```
(Существуют и другие dns сервера, например nameserver 77.88.7.7, если хотите можете вписать ниже через enter)
