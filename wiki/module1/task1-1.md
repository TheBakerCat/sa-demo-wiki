# 1. Произведите базовую настройку устройств:

## 1.1 Настройте имена устройств согласно топологии. 
Конфигурируем на всех хостах их hostname, используя полное доменное имя.

```shell
hostnamectl set-hostname "название машины"
```

::: tip
Полное доменное имя будет иметь вид:

```
%hostname%.%domain%
```

Где `%hostname%` - наименование машины из топологии, а `%domain%` наш домен. Пример:

```shell
hostnamectl set-hostname isp.au-team.irpo
```

Здесь, `isp` - наименование машины, `au-team.irpo` - домен.
:::

Перезапускаем оболочку

```shell
exec $SHELL
```


## 1.2 На всех устройствах необходимо сконфигурировать IPv4:
::: details
IP-адрес должен быть из приватного диапазона, в случае, если сеть
локальная, согласно RFC1918

* Локальная сеть в сторону HQ-SRV(VLAN 100) должна вмещать не
более 32 адресов

* Локальная сеть в сторону HQ-CLI(VLAN 200) должна вмещать не
менее 16 адресов

* Локальная сеть для управления(VLAN 999) должна вмещать не
более 8 адресов

* Локальная сеть в сторону BR-SRV должна вмещать не более 16
адресов

Сведения об адресах занесите в таблицу 2, в качестве примера
используйте Прил_3_О1_КОД 09.02.06-1-2026-М1
:::

Рассчитываем IP-адресацию устройств, заносим в таблицу.

На данном этапе гораздо логичнее рассчитать IP-адресацию сразу для всех машин и сетей, дабы пользоваться таблицей в дальнейшем.

Можно воспользоваться [подготовленным примером][1].
